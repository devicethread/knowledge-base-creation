This article covers all the tasks you should complete before connecting your Cloudbeds account with the Devicethread app.

Please make sure you have completed the following steps before connecting Cloudbeds and Devicethread.

See how to perform all four key steps in the following article here.

In summary:

  1. The primary user of your organization has downloaded the app from Devicethread's listing in the Cloudbed marketplace.

  2. You have set up one property: Name, Address, Users, etc. See how to set up Devicethread

  3. You have set up one or more SmartThings hubs for this property on your SmartThings app using your SmartThings account (our demo has two accounts and multiple hubs). See how to set up SmartThings app and hub(s).

  4. You have completed setting up all of the smart locks on the property and connected them to the SmartThings hub. See how to set up smart locks.

If you are done, then you are now ready to connect your Cloudbeds account and Devicethread.

1- Connecting Cloudbeds and Devicethread

  • Select the property on the portfolio page, and go to the "More" section of the app on the bottom left of the navigation bar.

  • Click "Cloudbeds" under the Integrations and Cloud accounts.

  • Click "Connect Cloudbeds" and follow the directions.
  • Enter your Cloudbeds credentials.
  • Verify your account by entering the one-time code sent to your email.

  • When your credentials have been validated, Devicethread goes to work instantly onboarding your property in the app:

    • Devicethread creates a "Collection" for each of the existing rooms on the property.

    • Key lock codes are created for each of the guests currently staying at the property.

  • Reconfirm the Cloudbeds property you wish to connect with the Devicethread account.


    Congratulations, you have now connected Cloudbeds with Devicethread successfully!

  • Now it is time to set up Devicethread based on your preferences;

    • Data Update Interval: Select the interval at which Devicethread will sync and update the appropriate information in Cloudbeds. Select from 15 minutes to 60 minutes and click "Apply".

    • Go to the "More" section and click "Collections".

    • You will recognize that all of the rooms/accommodations using the abbreviation name in Cloudbeds have been added as a single "Collection".


    The last step of the setup is to define which specific "Zoned and locks" each guest staying in a specific room/accommodation will have access to on the property. This is done in Devicethread by assigning all the appropriate "Zones" to each specific "Collection". Remember, you have already set up the Zones for the property before connecting the integration. if you have not done this yet, you can do it before continuing.

    For example, a guest staying in Room 101 should be able to open the Room 101 door lock, the Lobby front door lock, the backdoor lock, and the fitness room. In this example, a guest staying in Room 101 would receive a 4 or 6-digit lock code that will work in any of those 4 door locks and have access to these rooms during his stay at the property. After they check out, their codes will be expired and no longer work.

  • The following task will have to be repeated for each "Collection". Do not worry; this needs to be done only once when the integration is first connected. Also, you will be able to change these settings for each "Collection" at any time in the future:

    1. Select a specific "Collection" from the list.

    2. Click "Add Zones".

    3. Now you can see the list of "Zones". Remember, these are all of the "Zones" you have created previously. Each zone has one or more door locks (devices) that you associated with it.

  • Select all the property "Zones" by checking the boxes of the "Zones" a guest staying in that Room/Accommodation will have access to during their stay. Check all the appropriate boxes.

  • Click "Assign Selected Items".

    • Now repeat this step for every single "Collection" on the list.

  • After completing this, Voila! You are done with the initial setup and can now go back to the Cloudbeds interface. From now on, Devicethread will work in the background for you.

    Cloudbeds is now connected and configured

    • From now on, any new "Confirmed" or "In House/Checked in reservations" in your Cloudbeds account will automatically, within 1 or 2 minutes, have a lock code appear on the top panel of the reservation details page. To view the lock code click "Show More". Of note, a red dot appears beside the "Show More" button when a lock code is present.

    • If you set up a guest auto check-in email or text with room details, make sure to edit the template and add an "Insert Merge Tag" called "lockcode". The guest property access lock code will be inserted automatically in the check-in communication.





2- Disconnecting Cloudbeds account from Devicethread

  • Go to the "More" section of the app on the bottom left of the navigation bar.

  • Click "Cloudbeds" under the Integrations and Cloud accounts.

  • Click on the three dots menu on the top right, then click "Delete".

  • Click "Confirm".

  • Your Cloudbeds account has now been disconnected successfully:

    • All imported data from Cloudbeds has been deleted from your Devicethread account.

    • The initial property setup data created in the Devicethread app, before connecting to Cloudbeds, remains in the Devicethread app.
